# Массивы в C/C++

## Важное предупреждение

> ⚠️ **Компилятор не следит за границами массива.**
> При работе с массивами программист сам несёт ответственность за выход за их пределы.
> Доступ к элементам вне диапазона приводит к неопределённому поведению (undefined behavior).

---

## Объявление массивов

### Использование констант для размера массива

Для лучшей читаемости и удобства поддержки кода рекомендуется использовать именованную константу при объявлении массива:

```c
#define SIZE 10
// или
const int SIZE = 10;

int arr[SIZE];
```

### Защита массива от модификации

Для защиты массива от модификации использовать `const`:

```c
const int days[MONTHS];
```

Указатели на константы не могут использоваться для изменения значений.

## Многомерные массивы

### Объявление и инициализация

```c
int zippo[4][2];
```

### Адресация и указатели

- `zippo[0]` и `zippo` имеют одинаковые значения - начинаются в одной позиции
- `zippo[0]` - это массив из 2 целых чисел, эквивалентен `&zippo[0][0]`
- `zippo[0]` - адрес объекта типа `int`
- `zippo` - адрес объекта размером 2 значения типа `int`

### Арифметика указателей

При добавлении 1 к указателю или адресу получаем значение, которое больше исходного на размер указанного объекта. В этом отношении `zippo` и `zippo[0]` отличаются друг от друга:

- `zippo + 1` - указывает на следующий массив из 2 элементов
- `zippo[0] + 1` - указывает на следующий элемент в текущем массиве

### Разыменование указателей

- `*(zippo[0])` == `zippo[0][0]` -> значение типа `int`
- `zippo` - это адрес адреса (указатель на указатель), для получения обычного значения потребуется двухкратное разыменование

**Двойная косвенность** - адрес адреса или указатель на указатель.
