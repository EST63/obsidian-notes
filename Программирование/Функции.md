# Функции в языке C

## Ввод данных: scanf()

### Особенности использования scanf()

1. **Проблема с символом новой строки**
   
   После ввода числа и нажатия Enter в буфере ввода остаётся символ `\n`. Если затем читать символ (`%c`), функция может считать этот `\n`, а не ожидаемый символ. Эта проблема часто упоминается в учебниках.

2. **Проверка успешности ввода**
   
   Функция `scanf` возвращает количество успешно считанных значений. Рекомендуется проверять это значение, чтобы предотвратить сбои программы при неверном вводе.

3. **Очистка буфера**
   
   Иногда необходимо очистить буфер ввода от лишних символов (например, после `scanf` с `%d` и перед `%c`). В стандартной библиотеке C нет простой функции для этого, но можно использовать цикл с `getchar()`.

### Принцип работы scanf()

Функция `scanf` должна изменить переменную, записав в неё введённое значение. Для этого ей нужен адрес переменной в памяти, чтобы положить данные точно в нужное место. Поэтому перед именем переменной ставится оператор `&` (взятие адреса).

### Чтение строк с пробелами

Чтобы считать строку вместе с пробелами в `scanf()`, используется спецификатор формата **`%[^\n]`**:

```c
char name[50];
scanf("%[^\n]", name); // прочитает всё до нажатия Enter
```

Второй и более правильный способ — использование функции `fgets()`:

```c
char name[50];
fgets(name, sizeof(name), stdin); // безопасное чтение строки
```

## Вывод данных: printf()

Функция **`printf`** выводит значения на экран. Ей нужны только сами значения, поэтому переменные передаются **по значению** (в виде их копий).

## Работа со строками: strlen()

Функция **`strlen()`** возвращает длину строки в символах (не считая нуль-терминатора `\0`).

## Работа с пределами типов данных

- **`limits.h`** — определяет пределы для целочисленных типов (размеры `int` и др.)
- **`float.h`** — определяет пределы для типов с плавающей точкой

## Символьный ввод-вывод: putchar(), getchar()

- **`getchar()`** — возвращает символ из потока ввода. 
  Пример: `ch = getchar()` — эквивалентно `scanf("%c", &ch)`
- **`putchar()`** — выводит переданный аргумент.
  Пример: `putchar(ch);`

Эти функции работают с типом `char` и являются эффективными для посимвольного ввода-вывода.

---

## Анализ символов: ctype.h

Заголовочный файл **`ctype.h`** предоставляет набор функций для анализа и преобразования символов (например, `isalpha()`, `isdigit()`, `toupper()` и др.).

## Работа со строками: puts(), gets(), fgets()

- **`puts()`** — выводит символьную строку и автоматически добавляет символ новой строки `\n`.
- **`gets()`** — небезопасная функция, не выполняет проверку на переполнение буфера. Если вводимая строка слишком длинная, происходит переполнение. 
- **`fgets()`** — безопасная альтернатива `gets()`. Принимает три аргумента:
  - указатель на буфер
  - максимальное количество символов для чтения
  - поток ввода (для клавиатуры — `stdin`)
  
  Функция `fgets()` сохраняет символ новой строки в буфере, если он помещается.
- **`fputs()`** — запись строки, принимает два аргумента, второй указывает, куда производить запись (для вывода на экран — `stdout`).
- **`gets_s()`** — безопасная версия `gets()`, читает из стандартного ввода, не сохраняет символ новой строки.
- **`fputs()`** — принимает второй аргумент, указывающий на файл, куда производить запись (для вывода на экран — `stdout`), не выводит символ новой строки.
- strlen() - длинна строки
- strcat() - склеивание строк
- strncat() - склеивает строки и принимает кол-во символов
- strcmp() - сравнивание строк, строки а не символы и не указатели
